# SQL 工具箱手册

本目录存放用于 `zvt` 数据库运维、结构调整及性能优化的 SQL 脚本。由于 `zvt` 默认使用 SQLite 存储海量行情和逻辑数据，这些脚本在处理大规模数据维护时非常高效。

## 目录文件说明

| 文件名                   | 核心功能                                                                                                 | 适用对象                           |
| :----------------------- | :------------------------------------------------------------------------------------------------------- | :--------------------------------- |
| `reduce_size.sql`        | **数据瘦身与优化**。通过删除大量历史 K 线数据的重叠索引并压缩数据库空间，可将文件大小减小到原来的 1/10。 | 行情表 (如 `eastmoney_trading.db`) |
| `change_indices.sql`     | **表结构迁移与修正**。用于升级数据库架构、删除过时约束或修正历史数据中的拼写错误。                       | 元数据表 (如 `indices.db`)         |
| `change_stock_index.sql` | **股票及指数映射调整**。处理股票、指数等基础实体的元数据变更。                                           | 元数据表                           |
| `samples.sql`            | **初始化样本数据**。用于快速测试或环境搭建的示例 SQL 语句。                                              | 新测试库                           |

## 如何运行这些脚本

### 方法 1：命令行工具 (最快)
确保你的系统中已安装 `sqlite3`。

**基本语法：**
```bash
sqlite3 [数据库文件路径] < [SQL脚本文件路径]
```

**实战示例 (数据瘦身)：**
```bash
# 在项目根目录下执行
# 将 eastmoney 的交易数据库大小由 GB 级压缩至 MB 级
sqlite3 /data/code/zvt/zvt-home/data/eastmoney/eastmoney_trading.db < sql/reduce_size.sql
```

### 方法 2：使用 IDE 数据库控制台 (最直观)
*   **PyCharm**: 
    1. 在右侧 `Database` 面板点击 `+` -> `Data Source` -> `SQLite`。
    2. 找到对应的 `.db` 文件。
    3. 在打开的 `Query Console` 中直接复制脚本内容运行。
*   **VS Code**: 
    1. 安装插件 `SQLite Viewer` 或 `SQLTools`。
    2. 右键点击数据库文件选择打开，粘贴并执行 SQL 指令。

### 方法 3：使用图形化 GUI 工具
推荐使用 [DB Browser for SQLite](https://sqlitebrowser.org/)：
1. 打开对应的 `.db` 文件。
2. 切换到 `Execute SQL` (执行 SQL) 选项卡。
3. 点击“打开文件”图标导入本目录下的脚本。

## 🚨 重要安全提示

1.  **务必备份**：在运行任何 `DROP` 或 `CHANGE` 操作前，请先手动备份你的 `.db` 文件。
2.  **停止运行 zvt**：操作数据库时，请关闭正在运行的 `zvt_daily_job.py` 或 `zvt_server.py`，否则可能会因“Database Locked (数据库锁定)”报错。
3.  **索引重建**：执行 `reduce_size.sql` 删除索引后，虽然体积减小，但查询效率会降低。下次启动 `zvt` 采集任务时，框架会自动检测并重建必要的索引，此过程可能较慢。
